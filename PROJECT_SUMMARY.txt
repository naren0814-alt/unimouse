# Project Summary - Mesh Control

## What Was Built

A complete Windows-to-Windows multi-computer control application written in Python that allows one computer (server) to seamlessly control up to 2 other computers (clients) using a single keyboard and mouse.

## Key Features Implemented

✅ **Automatic Client Discovery**
- Clients broadcast presence every 2 seconds via UDP
- Server automatically discovers clients on same network
- No manual IP configuration required
- Stale client auto-removal (10-second timeout)

✅ **Seamless Cursor Edge Switching**
- Right edge of server screen → Client 1
- Right edge of Client 1 → Client 2
- Left edge of client → Back to server
- 5-pixel edge threshold for detection

✅ **Low-Latency UDP Networking**
- Discovery: Port 5050
- Control: Port 6060 (client receive), 6061 (return control)
- Compact packet format for minimal overhead
- Delta-based mouse movement for efficiency

✅ **Windows Native Input Injection**
- ctypes-based Windows API (SendInput/keybd_event)
- No PyAutoGUI dependency
- Full support for mouse movement and clicks
- Keyboard key injection with VK code mapping

✅ **Complete Tkinter GUI**
- Mode selection (Server/Client)
- Real-time client discovery list
- Connection status display
- Control indicator showing current active computer
- Start/Stop/Connect buttons
- Responsive design with proper state management

✅ **Multi-Computer Control (Up to 2 Clients)**
- Server controls keyboard and mouse
- Automatic state management
- Support for both client 1 and client 2
- Smooth transitions between computers

✅ **Input Capture with pynput**
- Mouse movement tracking (with delta calculation)
- Mouse click detection (left/right/middle)
- Keyboard press/release events
- Non-blocking event listeners in background threads

✅ **Client-Side Cursor Monitoring**
- Background thread monitors cursor position
- Detects left edge transitions
- Sends return control signal to server
- Properly throttled to avoid excessive packets

✅ **Robust Error Handling**
- Network error recovery
- Malformed packet handling
- Graceful socket cleanup
- Comprehensive logging system
- Daemon threads for automatic cleanup

## How It Works

1. **Server starts** → Listens for client broadcasts on UDP 5050
2. **Clients start** → Broadcast their hostname and IP every 2 seconds
3. **Server discovers** → Builds list of available clients
4. **User connects** → Server establishes control channel to client (UDP 6060)
5. **User moves cursor** → Captured by pynput, forwarded via network
6. **Client receives** → Injects input locally, cursor moves
7. **Cursor at edge** → State machine switches control
8. **Return control** → Client sends return signal when reaching left edge

## Network Communication

**Discovery Packet (Client → Server)**
```json
{
  "type": "mesh_client",
  "hostname": "LAPTOP-ABC123",
  "ip": "192.168.1.100"
}
```

**Control Packets (Server → Client)**
```json
["m", 15, -8]           # Mouse move: 15px right, 8px up
["c", "left"]           # Left click
["k", "a", true]        # Key 'a' pressed down
["k", "a", false]       # Key 'a' released
```

**Return Control (Client → Server)**
```json
{"type": "return_control"}
```

## Installation & Usage

**Install Dependencies:**
```powershell
pip install pynput
```

**Run Application:**
```powershell
python mesh_control_gui.py
```

**Server Setup:**
1. Start application, select "Server"
2. Click "Start Server"
3. Select clients from list
4. Click "Connect Selected" for each client

**Client Setup:**
1. Start application on each client
2. Select "Client"
3. Click "Start Client"
4. Client broadcasts to server automatically

**Use:**
- Move cursor to right edge to switch to client
- Move to left edge to return to server
- Keyboard and mouse work automatically
